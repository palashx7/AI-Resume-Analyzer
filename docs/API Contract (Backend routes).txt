API Contract — AI Resume Analyzer & Job Match Platform
✅ Base URL

During local development:

http://localhost:8000/api

Production (example):

https://your-backend-domain.com/api

All protected routes require:
✅ Authorization: Bearer <JWT_TOKEN>

1) ✅ Auth APIs
1.1 Register User
POST /auth/register

Request Body

{
  "name": "Palash Bhivgade",
  "email": "palash@gmail.com",
  "password": "StrongPassword123"
}


Response (201 Created)

{
  "message": "User registered successfully"
}


Errors

400 Bad Request → missing fields

409 Conflict → email already registered

1.2 Login User
POST /auth/login

Request Body

{
  "email": "palash@gmail.com",
  "password": "StrongPassword123"
}


Response (200 OK)

{
  "token": "JWT_TOKEN_HERE",
  "user": {
    "id": "userObjectId",
    "name": "Palash Bhivgade",
    "email": "palash@gmail.com",
    "role": "user"
  }
}


Errors

401 Unauthorized → invalid credentials

1.3 Get Current User Profile
GET /auth/me ✅ Protected

Response (200 OK)

{
  "id": "userObjectId",
  "name": "Palash Bhivgade",
  "email": "palash@gmail.com",
  "role": "user"
}

2) ✅ Resume APIs
2.1 Upload Resume PDF
POST /resumes/upload ✅ Protected

Request Type: multipart/form-data

Form Data

file → Resume PDF

resumeTitle → optional string

Response (201 Created)

{
  "message": "Resume uploaded successfully",
  "resume": {
    "id": "resumeObjectId",
    "resumeTitle": "Resume V1",
    "originalFileName": "palash_resume.pdf",
    "extractedTextPreview": "First 300 chars...",
    "createdAt": "2026-01-15T10:30:00Z"
  }
}


Validation Rules

File type must be PDF

Max size: 2MB

Extracted text must not be empty

Errors

400 Bad Request → invalid file type/size

422 Unprocessable Entity → text extraction failed

2.2 Get All User Resumes
GET /resumes ✅ Protected

Response (200 OK)

{
  "resumes": [
    {
      "id": "resumeObjectId",
      "resumeTitle": "Resume V1",
      "originalFileName": "resume.pdf",
      "createdAt": "2026-01-15T10:30:00Z"
    }
  ]
}

2.3 Get Resume Details
GET /resumes/{resumeId} ✅ Protected

Response (200 OK)

{
  "resume": {
    "id": "resumeObjectId",

    "originalFileName": "resume.pdf",
    "extractedText": "Full extracted resume text...",
    "createdAt": "2026-01-15T10:30:00Z"

4) ✅ Analysis APIs (Core)
4.1 Run Analysis
POST /analysis/run ✅ Protected

Request Body

{
  "resumeId": "resumeObjectId",
  "jobDescriptionId": "jdObjectId"
}


Response (200 OK)

{
  "analysisId": "analysisObjectId",
  "scores": {
    "atsScore": 76,
    "similarityScore": 68
  },
  "keywordMatch": {
    "matchedSkills": ["Python", "REST API", "MongoDB"],
    "missingSkills": ["Docker", "AWS", "Redis"]
  },
  "strengths": [
    "Your resume contains relevant backend keywords and project experience."
  ],
  "improvements": [
    "Add missing skills mentioned in the JD if you have worked with them.",
    "Include measurable outcomes in project bullet points."
  ],
  "resumeTips": [
    {
      "title": "Improve Project Bullet",
      "suggestion": "Built a REST API using FastAPI and MongoDB to support resume-job matching with 70% faster search."
    }
  ],
  "createdAt": "2026-01-15T11:30:00Z"
}


Validation Rules

resumeId must exist and belong to logged-in user

jobDescriptionId must exist and belong to logged-in user

Resume extractedText should not be empty

JD text should not be empty

Errors

404 Not Found → resume or JD not found

422 Unprocessable Entity → analysis failed
