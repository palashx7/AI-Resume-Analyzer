4) AI / NLP Design (Free & Practical)
4.1 Extract Text

Resume PDF → extractedText (string)

JD text → cleanedText (string)

4.2 Preprocessing

lowercase

remove extra spaces

remove special characters (optional)

basic stopwords removal (optional)

4.3 ATS Keyword Match Score

Method:

Extract keywords/skills from JD

Check presence in resume text

Compute score = matched / required * 100

Outputs:

matchedSkills

missingSkills

atsScore

4.4 Similarity Score (AI)

Method:

Use sentence-transformers embeddings

cosine similarity between resume text and JD text

convert similarity into 0–100 range

Outputs:

similarityScore

4.5 Suggestions Engine

Rule-based improvements like:

Add missing skills (if relevant)

Improve bullet point format

Add measurable outcomes (numbers)

Outputs:

strengths

improvements

resumeTips[]

5) Data Flow (Request Lifecycle)
Step-by-step flow

User logs in → receives JWT token

Upload resume PDF → backend extracts text and stores it

User adds JD → backend stores it

User runs analysis → backend:

pulls resume text + JD text

performs ATS scoring + similarity scoring

creates suggestions

saves analysis result in DB

Results displayed on frontend

User can revisit history anytime

6) Security & Validation (Must follow)
